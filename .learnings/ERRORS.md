# Errors

開発中に捕捉したコマンドの失敗、APIエラー、予期しない挙動。

**Areas**: frontend | backend | infra | tests | docs | config
**Statuses**: pending | in_progress | resolved | wont_fix | promoted | promoted_to_skill

## ステータスの定義

| Status | 意味 |
|--------|---------|
| `pending` | 未対応 |
| `in_progress` | 現在対応作業中 |
| `resolved` | 問題が修正済み（Resolution ブロックを追加） |
| `wont_fix` | 対応しないと判断（理由を Resolution に記載） |
| `promoted` | CLAUDE.md、AGENTS.md、または copilot-instructions.md へ昇格済み |
| `promoted_to_skill` | 再利用可能なスキルとして抽出済み |

エントリ形式: self-improvement スキルの「Error Entry」セクションを参照。ID は `ERR-YYYYMMDD-XXX` を使う。
