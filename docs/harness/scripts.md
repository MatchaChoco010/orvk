# エージェント向けスクリプトと一時検証スクリプトの扱い

このドキュメントは **スクリプトを書く/置く場所を決めるときに読む**。

## 再利用するスクリプトは scripts/ に Node.js で

AI エージェントが使う skill やハーネスの**再利用**スクリプト(skill のフックやヘルパーを含む)は **`scripts/` 以下**に置き、**Node.js** で実装する。
Python だとグローバルにパッケージを入れられず `.venv` を毎回 activate する必要があるが、Node なら `scripts/package.json` と `scripts/node_modules/` を置いておけば activate 不要で即実行できる。
Windows(PowerShell 主体)でも摩擦が少なく、Claude Code ハーネスや skill の hook も JS/TS 系で一貫する。
`jq` や `python3` への依存は、環境によって(とくに Windows)不在・破損するため避ける。
**今後 scripts/ は Node で揃える。**
詳細は [scripts/README.md](../../scripts/README.md)。

skill に同梱するフックスクリプトのように、特定 skill に強く結びつくものは、その skill 配下(`.claude/skills/<skill>/scripts/`)に併置してよい。
その場合も Node で書き、純 Node stdlib(依存ゼロ)に保って OS 非依存にする。

## 一時的な検証スクリプトはリポジトリに残さない

動作確認や調査のためにその場で書く**使い捨てスクリプト**は、リポジトリにコミットしない。
**スクラッチパッド(セッション専用の作業ディレクトリ)** などリポジトリ外で書いて使い、終わったら捨てる。
リポジトリに置いてよいのは「何度も使う」と確信できるものだけで、その場合は再利用手段(README・呼び出し元 skill)ごと scripts/ に整備する。

一時検証スクリプトと再利用スクリプトを混同しない。
判断基準:

- アサーション(期待値の機械判定)が無く、出力を人間/エージェントが目視で確かめるだけ → 一時検証。リポジトリに残さない。
- 回帰を継続的に守る必要があるもの → 一時スクリプトではなく **Rust のテスト**(`cargo test`)にする。
- 何度も繰り返すワークフローの手順 → scripts/ に Node で整備する。
