# Feature Requests

開発中に捕捉した、ユーザーから要望された欠けている機能。

**Areas**: frontend | backend | infra | tests | docs | config
**Statuses**: pending | in_progress | resolved | wont_fix
**Complexity**: simple | medium | complex

## ステータスの定義

| Status | 意味 |
|--------|---------|
| `pending` | 未対応 |
| `in_progress` | 現在構築中 |
| `resolved` | 機能を実装済み（Resolution ブロックを追加） |
| `wont_fix` | 構築しないと判断（理由を Resolution に記載） |

エントリ形式: self-improvement スキルの「Feature Request Entry」セクションを参照。ID は `FEAT-YYYYMMDD-XXX` を使う。
